# サンドボックスセキュリティテスト

このドキュメントは、Claude Code のサンドボックス設定とフックが正しく機能しているか確認するためのテストガイドです。

## クイックテスト

Claude Code で以下のように依頼してください：

```
サンドボックスセキュリティのテストを実行して。以下を確認：
1. ホームディレクトリ (~/) へのアクセスがブロックされるか（ls ~/, cd ~, echo ~ など）
2. Git の HEAD~3 などのチルダ記法が誤検知されないか
3. 親ディレクトリ (../) へのアクセスがブロックされるか
4. 複合コマンド（git status && ls ..）での親ディレクトリアクセスがブロックされるか
5. シンボリックリンク経由のアクセスがブロックされるか
6. 作業ディレクトリ内のファイルは読めるか
7. Git セキュリティ（remote add, 外部clone, origin以外へのpush）がブロックされるか
```

## 詳細テストケース

### 1. ホームディレクトリアクセステスト

#### テスト 1-1: Bash でのホームディレクトリアクセス

```bash
cat ~/.gitconfig
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-2: Read ツールでのホームディレクトリアクセス

```
Readツールで ~/.gitconfig を読み取る
```

**期待結果**: ❌ ブロック（フックがサンドボックス許可設定より優先される）

#### テスト 1-3: 絶対パスでのホームディレクトリアクセス

```bash
cat /Users/$(whoami)/.gitconfig
```

**期待結果**: ❌ サンドボックスまたはフックでブロック

#### テスト 1-4: ls ~ でのホームディレクトリ一覧

```bash
ls ~/
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-5: cd ~ でのホームディレクトリ移動

```bash
cd ~
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-6: echo ~ でのチルダ展開

```bash
echo ~
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-7: cd ~ && pwd での複合コマンド

```bash
cd ~ && pwd
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-8: ls ~/Documents でのホームサブディレクトリ

```bash
ls ~/Documents
```

**期待結果**: ❌ `ホームディレクトリへのアクセスは禁止されています`

#### テスト 1-9: Git の HEAD~3 記法（誤検知防止テスト）

```bash
git log HEAD~3 --oneline
```

**期待結果**: ✅ 正常に実行される（`~` を含むが、git参照として許可される）

#### テスト 1-10: Git 範囲指定でのチルダ使用

```bash
git diff HEAD~2..HEAD --stat
```

**期待結果**: ✅ 正常に実行される

#### テスト 1-11: ブランチ参照でのチルダ使用

```bash
git show main~1 --stat
```

**期待結果**: ✅ 正常に実行される

### 2. 親ディレクトリアクセステスト

#### テスト 2-1: Bash での親ディレクトリアクセス

```bash
cat ../README.md
```

**期待結果**: ❌ `親ディレクトリへのアクセスは禁止されています`

#### テスト 2-2: Read ツールでの親ディレクトリアクセス

```
Readツールで ../README.md を読み取る
```

**期待結果**: ❌ ブロックされる

#### テスト 2-3: Git 範囲指定（例外ケース）

```bash
git log main..HEAD
```

**期待結果**: ✅ 正常に実行される（.. は範囲指定演算子として許可）

#### テスト 2-4: 複合コマンドでの親ディレクトリアクセス（&&）

```bash
git status && ls ..
```

**期待結果**: ❌ `親ディレクトリへのアクセスは禁止されています`
**理由**: git コマンドを含む複合コマンドでも、git以外の部分に`..`が含まれる場合はブロックされる

#### テスト 2-5: 複合コマンドでの親ディレクトリアクセス（||）

```bash
git log || cd ..
```

**期待結果**: ❌ `親ディレクトリへのアクセスは禁止されています`

#### テスト 2-6: 複合コマンドでの親ディレクトリアクセス（;）

```bash
git status; rm -rf ../test.txt
```

**期待結果**: ❌ `親ディレクトリへのアクセスは禁止されています`

#### テスト 2-7: Git 範囲指定を含む複合コマンド（正当な使用）

```bash
git log HEAD..origin/main && echo "Done"
```

**期待結果**: ✅ 正常に実行される（git範囲指定は許可され、echoも問題なし）

#### テスト 2-8: Git 範囲指定と親ディレクトリアクセスの混在

```bash
git log HEAD..main && cd ..
```

**期待結果**: ❌ `親ディレクトリへのアクセスは禁止されています`
**理由**: git範囲指定の`..`は許可されるが、`cd ..`の`..`はブロックされる

### 5. ネットワークアクセスの制御

#### 設定方針

このプロジェクトでは、ネットワークアクセスを**許可制**にしています。

```json
{
  "sandbox": {
    "network": {
      "allowedDomains": []
    }
  }
}
```

**動作**:

- すべてのネットワークアクセスでユーザーに許可を求めるプロンプトが表示される
- 情報検索などで必要な場合に、その都度許可できる
- 完全ブロックではなく、柔軟な制御を実現

**確認方法（任意）**:
Claude Code に「curl https://www.google.com を実行して」と指示すると、許可プロンプトが表示されることを確認できます。（自動テストには含めません）

#### テスト 6-2: origin 以外への git push のブロック

```bash

```

**期待結果**: ❌ `origin 以外のリモートへの git push は禁止されています`

#### テスト 6-3: 外部リポジトリの git clone のブロック

```bash

```

**期待結果**: ❌ `外部リポジトリからの git clone は禁止されています`

```bash

```

**期待結果**: ✅ 許可される（ただし dangerouslyDisableSandbox: true が必要）

## トラブルシューティング

### テストが失敗する場合

1. **フックファイルが存在するか確認**

   ```bash
   ls -la .claude/hooks/check-sensitive-paths.sh
   ```

2. **フックの内容を確認**
   - シンボリックリンクチェックの処理が実装されているか
   - ホームディレクトリの早期チェック処理が実装されているか
   - チルダ（~）とgit参照を区別する正規表現パターンが使用されているか

3. **サンドボックスモードが有効か確認**

   ```bash
   echo $TMPDIR
   ```

   出力が `/tmp/claude/` であれば有効

4. **フックが実行可能か確認**
   ```bash
   chmod +x .claude/hooks/check-sensitive-paths.sh
   ```

### 意図しないブロックが発生する場合

- `.claude/hooks/check-sensitive-paths.sh` のホワイトリストを確認
- `WORK_DIR` 変数が正しく設定されているか確認
- フックのログを確認（デバッグモードを有効化する場合）

## 導入先プロジェクトでの使用方法

1. `.claude/` ディレクトリをプロジェクトにコピー
2. このドキュメント（SANDBOX-TEST.md）をプロジェクトルートに配置
3. Claude Code で「SANDBOX-TEST.md を読んでテストを実行して」と指示

すべてのテストが期待通りの結果になれば、サンドボックスセキュリティは正しく設定されています。
