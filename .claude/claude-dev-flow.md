# Claude Code 開発フロールール

このファイルはClaude Codeが参照するための開発フロー指示書です。
以下のルールをすべてのタスクで**必ず**遵守すること。

---

## 基本原則

- 仕様に不明点があれば、推測で進めず必ず開発者に質問する
- 開発者の明示的な承認なしに、ファイルの書き込みやコード変更を行わない
- 実装後は必ず関連するテストを実行し、パスすることを確認する

---

## 開発の3ステップ・ループ

### ステップ1：計画（Plan）

実装前に以下を行い、開発者の承認を得ること。

1. 既存コードを調査し、影響範囲を把握する
2. 不明点をすべて開発者に確認する
3. 以下の内容を含む詳細設計をまとめ、承認を得る
   - 変更・追加するファイルの一覧
   - 各ファイルへの変更内容（関数・クラス・ロジックの変更点）
   - 既存の実装で流用できる箇所
   - 懸念点やリスク

**完了条件（すべて満たしてから次へ進む）:**
- [ ] 実装の目的・背景が明確になっている
- [ ] 既存の実装を調査し、影響範囲を把握している
- [ ] 仕様の不明点がすべて解消されている
- [ ] 詳細設計について開発者の承認を得た

承認を得たら必ず確認する:
> 「ステップ1の完了要件をすべて満たしていることを確認しました。ステップ2（実装）に進んでよいですか？」

---

### ステップ2：実行（Do）

開発者から明示的なGOサインを受けてから実装を開始すること。

- 詳細設計の範囲外のファイルは変更しない
- 実装完了後、関連するテストを実行する
- テストがパスしたら開発者にレビューを依頼する
- 指摘があれば修正し、再度テストを実行してから再レビューを依頼する

**完了条件（すべて満たしてから次へ進む）:**
- [ ] すべての変更が詳細設計の内容に沿っている
- [ ] テストがすべてパスしている
- [ ] エラー・警告が解消されている
- [ ] 想定外の変更（設計外のファイル編集など）が発生していない
- [ ] 開発者のレビューで承認を得た

承認を得たら必ず確認する:
> 「ステップ2の完了要件をすべて満たしていることを確認しました。ステップ3（まとめ）に進んでよいですか？」

---

### ステップ3：まとめ（Wrap-up）

以下を作成して開発者に最終確認を求める。

- 今回の変更内容の箇条書きまとめ
- PRのタイトルと概要

**完了条件（すべて満たしてから作業完了とする）:**
- [ ] 変更内容が箇条書きでまとめられている
- [ ] PRのタイトルと概要が作成されている
- [ ] 開発者が最終確認（論理的なバグ・仕様漏れがないか）を行った

> 「ステップ3の完了要件をすべて満たしました。最終確認をお願いします。問題なければ作業完了です。」
